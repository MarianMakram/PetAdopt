Project Overview

منصة بتربط المحميات وأصحاب الحيوانات بالناس اللي عايزة تتبنى. الفكرة إنك تعمل Bridge بين كل الأطراف — Adopter بيدور، Owner بيعرض، Admin بيراجع.

⚙️ .NET Core Web API (C#) ⚛️ React.js + Vite + Tailwind 🗄️ SQL Server + EF Core (Code First) ⚡ SignalR (WebSocket)

أنواع المستخدمين (Actors)

👑

Admin

يدير المستخدمين، يوافق/يرفض الحسابات والبوستات

🏠

Shelter / Pet Owner

يعرض حيوانات للتبني، يدير الطلبات الواردة عليه

💛

Adopter

يتصفح، يفضّل، يطلب تبني، يكتب ريفيو بعد التبني

05

Functional Requirements

01أنواع المستخدمين المختلفة يقدروا يسجلوا دخول/خروج. الـ Adopter يتصفح بدون تسجيل دخول بس مينقدرش يطلب تبني من غيره.
02الـ Admin يقدر يوافق أو يرفض حسابات جديدة (Shelters أو Pet Owners).
03الـ Admin يوافق أو يرفض بوستات الحيوانات قبل ما تظهر للعموم.
04المتبنيون يتصفحوا ويبحثوا بالنوع والسلالة والسن والموقع.
05البوست يشمل: اسم المالك، اسم الحيوان، السن، السلالة، الجنس، الحالة الصحية، الوصف، الموقع، الصور.
06المتبنيون يقدروا يقدموا طلبات تبني.
07أصحاب الحيوانات يقبلوا أو يرفضوا طلبات التبني.
08بعد القبول، حالة الحيوان تتغير لـ Adopted تلقائياً.
09المتبنيون يحفظوا حيوانات في قائمة المفضلة.
10اللي اتبنوا بنجاح يقدروا يكتبوا ريفيو وتقييم للمحمية أو المالك.
11Shelter/Pet Owner يدير بوستاته: إنشاء، عرض، تعديل، حذف (CRUD).
12لازم يستخدم Real-Time Sockets للإشعارات (طلبات التبني على الأقل).
13تصميم قاعدة البيانات (Schema) مطلوب.

Real-Time SignalR Events

🔌 Hub: NotificationHub 🛣️ Route: /hubs/notifications 🔐 Auth: JWT in query string 👤 Infrastructure: A1 + B1 (DB) · Hub setup: A1
admin:new_shelter_registration New shelter/owner registered, needs admin review → role:admin
admin:new_pet_post New pet post submitted, needs admin review → role:admin
admin:post_moderated Admin approved/rejected a pet post → userId:[ownerId]
shelter:registration_status_changed Admin approved/rejected shelter registration → userId:[shelterId]
adoption:new_request New adoption request for a pet → userId:[ownerId]
adoption:request_status_changed Owner accepted/rejected an adoption request → userId:[adopterId]
review:new_review New review submitted for a pet → userId:[ownerId]

Client-Side Listeners

🟢 Team A (Adopter side)

يسمع لـ adoption:request_status_changed → يحدث بادج الطلب + Toast

🔵 Team B (Owner/Admin)

يسمع لـ adoption:new_request · admin:new_shelter_registration · admin:new_pet_post
