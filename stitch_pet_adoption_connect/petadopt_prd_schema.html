# PetAdopt Platform - Product Requirements & Database Schema ## Project Overview PetAdopt is a comprehensive platform designed to facilitate the adoption of pets by connecting animal shelters and individual pet owners with potential adopters. The platform supports three distinct user roles: Admins, Owners/Shelters, and Adopters. ## User Roles & Permissions ### 1. Adopter * **Unauthenticated:** Browse pets, search, and view pet details. * **Authenticated:** * Apply for adoption. * Save pets to favorites. * Manage adoption requests ("My Requests"). * Submit reviews/feedback for owners/shelters after successful adoption. ### 2. Owner / Shelter * Manage pet listings (CRUD: Create, Read, Update, Delete). * View and manage (Accept/Reject) adoption requests via a dashboard. * Once an adoption is accepted, the pet status automatically updates to "Adopted". ### 3. Admin * Manage user registrations: Approve or reject new Shelter/Owner accounts. * Content Moderation: Approve or reject new pet listings before they go public. ## Technical Requirements * **Real-time Sockets:** Implementation of real-time notifications for adoption request status changes and new messages. * **Search/Filtering:** Advanced search by animal type, breed, age, and location. --- ## Database Schema (Proposed) ### Users Table * `id` (PK) * `email` (Unique) * `password_hash` * `role` (ENUM: 'admin', 'owner', 'adopter') * `full_name` * `phone_number` * `location` * `is_approved` (Boolean, default: false - for owners/shelters) * `created_at` ### Pets Table * `id` (PK) * `owner_id` (FK -> Users.id) * `name` * `type` (e.g., Dog, Cat, Rabbit) * `breed` * `age` * `gender` (ENUM: 'male', 'female') * `health_status` * `description` (Text) * `location` * `images` (JSON/Array of URLs) * `status` (ENUM: 'pending_approval', 'available', 'adopted') * `created_at` ### AdoptionRequests Table * `id` (PK) * `pet_id` (FK -> Pets.id) * `adopter_id` (FK -> Users.id) * `status` (ENUM: 'pending', 'accepted', 'rejected') * `message` (Text) * `created_at` * `updated_at` ### Favorites Table * `id` (PK) * `user_id` (FK -> Users.id) * `pet_id` (FK -> Pets.id) ### Reviews Table * `id` (PK) * `adopter_id` (FK -> Users.id) * `owner_id` (FK -> Users.id) * `rating` (Integer, 1-5) * `comment` (Text) * `created_at` --- ## Screen List to be Designed: 1. **Home:** Search and featured pets. 2. **Pet Details:** Comprehensive pet info and "Adopt" button. 3. **Login:** Unified login for all roles. 4. **Register:** Role selection and registration form. 5. **My Requests (Adopter):** List of sent adoption requests. 6. **Favorites (Adopter):** Saved pets. 7. **Add Review (Adopter):** Post-adoption feedback form. 8. **My Pets (Owner):** List of owner's pet posts. 9. **Add/Edit Pet (Owner):** Form for listing/updating pets. 10. **Requests Dashboard (Owner):** Management of incoming adoption requests. 11. **User Approval (Admin):** Reviewing new owner/shelter accounts. 12. **Pet Approval (Admin):** Reviewing new pet listings.
